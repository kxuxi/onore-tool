# ONORE ANALYTICS

Next.js + TypeScript 製の、ゲーム「己鯖」用の武将データベースツールです。
戦闘履歴を貼り付けるだけで攻撃側・防衛側の武将のタイプ・兵科を共有データベース（PostgreSQL）に登録し、偵察した武将名リストから即座に検索できます。武将名・兵種名はクリックで詳細ページ（戦績・勝率・対戦ログ）に展開できます。

## 機能

- **戦闘履歴タブ**: タブ区切りの戦闘履歴を貼り付けて「登録する」で両陣営の武将を一括登録（`Cmd`/`Ctrl` + `Enter` でも登録可）。並び替え（新しい順 / 古い順）・国での絞り込み・キーワード検索（一致部分をハイライト表示）に対応。登録に成功すると入力欄は自動でクリアされます
- **偵察検索タブ**: スペース / 改行 / カンマ区切りで武将名を貼り付けてタイプ・兵科・兵種を即座に表示。登録済 / 未登録の件数サマリーと入力クリアボタンを備え、国へ報告するための「名前［タイプ｜兵種］」形式のテキスト（件数つき）も生成・コピーできます。表の「結果をコピー」は見出し行と国を含む TSV で出力します
- **被弾表タブ**: 登録済みデータから各武将の行動状況（行動可 / 不明 / 行動済み）を集計表示。武将名での絞り込みに対応し、時刻は自動更新されます
- **武将ランキングタブ**: 出兵勝利数 / 守備勝利数 / SWI（攻撃）/ SWI（守備）の4指標で武将を順位表示（数値は桁区切り表示）。兵科・武将名・最低出撃数で絞り込み
- **DB確認タブ**: 登録済み武将の一覧 / 名前や国・タイプ・兵科・兵種で絞り込み。列ヘッダー（国・武将名・タイプ・兵科・兵種・行動時間・更新日時）で並び替え、結果を TSV でコピー可能
- **兵種図鑑タブ**: 兵種マスタ（攻撃・防御・コスト・特殊攻撃など）の閲覧・追加・編集
- **武器図鑑タブ**: 戦闘履歴の装備2（武器）から使用回数・勝率・主な使用武将を集計表示
- **品物図鑑タブ**: 戦闘履歴の装備1（品物）から使用回数・勝率・主な使用武将を集計表示
- **国カラータブ**: 勢力ごとの表示色を設定（この設定のみブラウザの `localStorage` に保存）
- **武将ページ / 兵種ページ**: 一覧や戦闘カード内の武将名・兵種名をクリックすると、戦闘数・勝率・対戦ログを表示する詳細ページへ。相互リンクと「戻る」スタックで自然に行き来できます
- **共有リンク / DB更新**: 表示中のタブ・詳細ページは URL に反映され、ヘッダーや詳細ページの「共有 / リンクをコピー」ボタン、再読込でそのまま復元できます。ロゴ（ONORE ANALYTICS）クリックでホーム（戦闘履歴）へ戻り、ヘッダーの「更新」ボタンで共有DBを手動再取得できます
- **スマホ / PWA 対応**: 狭い画面では表をカード表示に切り替え、ノッチ端末（safe-area）にも対応。Web App Manifest を提供し、ホーム画面へ追加してスタンドアロン起動できます
- **アクセシビリティ**: 本文へのスキップリンク、キーボードフォーカスの可視化、エラー通知の読み上げ（assertive）、長い一覧では「先頭へ戻る」ボタンを表示

武将DBと戦闘履歴はサーバー（Prisma + PostgreSQL）に保存され、複数の端末・ブラウザで共有されます。国カラー設定のみ各ブラウザの `localStorage` に保存されます。

## ローカルで動かす

データベースに PostgreSQL を使用します。接続先を環境変数 `DATABASE_URL` で指定してください。

1. 依存パッケージをインストール

   ```bash
   npm install
   ```

2. プロジェクト直下に `.env` を作成し、接続文字列を設定（雛形 `.env.example` をコピーすると簡単です）

   ```bash
   cp .env.example .env
   # .env を開いて DATABASE_URL を実際の接続文字列に書き換える
   DATABASE_URL="postgresql://USER:PASSWORD@HOST:5432/DBNAME"
   ```

3. マイグレーションを適用し、兵種マスタの初期データを投入

   ```bash
   npx prisma migrate deploy   # スキーマをDBへ反映
   npx prisma db seed          # 兵種マスタ(seed-unit-types.tsv)を投入
   ```

4. 開発サーバーを起動

   ```bash
   npm run dev
   ```

ブラウザで http://localhost:3000 を開いてください。

> Prisma の設定は `prisma.config.ts` に集約しています（`.env` は同ファイル内の `dotenv` で読み込まれます）。

## Vercel にデプロイ

1. このリポジトリを GitHub にプッシュ
2. [Vercel](https://vercel.com/) の "New Project" でリポジトリを Import
3. Framework Preset は自動で **Next.js** が選択されます
4. 環境変数 `DATABASE_URL` に本番 PostgreSQL の接続文字列を設定
5. Deploy（ビルド時に `prisma generate && next build` が実行されます）

> **マイグレーションはビルド時には実行しません。** Vercel のビルド環境からの直接DB接続に依存させず、デプロイを安定させるためです（Prisma Postgres の場合、Vercel 側の `DATABASE_URL` は Accelerate 経由のプール用URLになりがちで、`prisma migrate deploy` には直接接続URLが必要です）。
>
> スキーマを変更したときは、**直接接続できる `DATABASE_URL`** を設定したローカル等から本番DBに対して次を実行してください。
>
> ```bash
> npm run db:migrate   # = prisma migrate deploy
> npx prisma db seed   # 初回のみ：兵種マスタを投入
> ```

## 戦闘履歴のフォーマット

タブ区切りで以下の構造を想定しています:

```
【N戦目】<TAB>年月<TAB>時刻<TAB>場所<TAB>勢力名<TAB>武将名<TAB>家名<TAB>タイプ<TAB>兵種名<TAB>兵科<TAB>装備1<TAB>装備2<TAB>V.S.<TAB>勢力名<TAB>武将名<TAB>家名<TAB>タイプ<TAB>兵種名<TAB>兵科<TAB>装備1<TAB>装備2<TAB>勝敗<TAB>ターン数
```

- タイプ例: `武特` `統特` `知特` `武統` `統知` `知武` `政治家` `謎`
- 兵科例: `騎兵` `歩兵` `弓兵` `万能` `妖怪`

タブが含まれない場合は 2 文字以上の連続空白でもフォールバック分割します。

## データ構成と API

データは Next.js の API ルート経由で PostgreSQL に読み書きします（クライアントから直接DBへはアクセスしません）。

- `GET /api/state` — 武将DBと戦闘履歴を取得
- `POST /api/state` — 解析済みの武将・戦闘履歴を登録（戦闘履歴は重複排除キーで自動重複除外）
- `GET/POST /api/unit-types` — 兵種マスタの取得・追加/更新
- `DELETE /api/unit-types/[name]` — 兵種を削除

主なテーブル（`prisma/schema.prisma`）:

- `Warlord` — 武将（名前・勢力・タイプ・兵科・兵種・行動履歴など）
- `BattleRecord` — 戦闘履歴（`line` が重複排除キー、`raw` が表示・再解析用の元テキスト）
- `UnitType` — 兵種マスタ（攻撃・防御・コスト・特殊攻撃など）
